Want to know the secret to development? Learning to build software is more about learning how to think than it is about learning Python, Java or Ruby. The problem with programming is that there is never one right way to build a product. There are many approaches to every problem that will work, but some are better than others given the scope of the project. Dealing with this ambiguity has spawned brand new kinds of thinking that engineers discuss with each other all the time. We've been listening.
